# 2. 核心关系说明

## 2.1 用户与教师资料
- `users` 保存认证信息
- `teacher_profiles` 保存教师业务资料
- 一个用户对应一份教师资料

## 2.2 用户与班级
- 用户和班级是多对多关系
- 中间表 `class_teachers` 记录角色
- 角色至少包括：
  - `class_admin`
  - `class_teacher`

## 2.3 班级与学生
- 一个班级下有多个学生
- 学生必须归属一个班级

## 2.4 班级与小组
- 一个班级下有多个小组
- 小组成员通过 `group_members` 维护
- 一个学生首版只属于一个小组

## 2.5 班级与积分项目
- 每个班级拥有自己的积分项目配置
- 项目可带学科信息

## 2.6 积分日志
- 学生打分日志独立存 `student_score_logs`
- 小组打分日志独立存 `group_score_logs`
- 撤销采用“标记 + 撤销关联”的审计式设计

## 2.7 学期与班级
- 一个学期可关联多个班级
- 学期由教师手动创建

# 4. 组件拆分建议

## 通用组件
- TopNav 顶部导航
- SideMenu/DrawerMenu 侧边菜单/抽屉菜单
- ClassCard 班级卡片
- StudentCard 学生卡片
- GroupCard 小组卡片
- ScoreItemButton 快捷评分按钮
- StatChart 图表卡片
- EmptyState 空状态组件
- ConfirmDialog 确认弹窗

## 业务组件
- StudentScoreDialog 学生评分弹窗
- GroupScoreDialog 小组评分弹窗
- StudentImportPanel 学生导入面板
- ClassTeacherManager 教师关联管理
- TermSelector 学期选择器
- GrowthBadge 成长阶段展示组件

# 5. 页面开发优先级

## P0（最先做）

## P1

## P2
- 教师详情页

# 6. 需要最终拍板的小点
1. 登录字段是否固定用户名
2. 学生成长风格是否班级统一
3. 导入时小组不存在是否自动创建
4. 历史页默认展示学生记录还是全部记录
